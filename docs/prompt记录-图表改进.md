# Prompt 记录

## 原始 Prompt

> 针对论文中的图（主要看视觉效果），分析是否使用mermaid来画效果更好，首先准备出你的改进方案和升级对应关系，然后修改（你可以上网搜索学术论文在typst中怎么使用mermaid、以及有什么好用的skill）

## Prompt 结构拆解

```
[目标对象] 论文中的图
[核心诉求] 视觉效果优化
[指定工具] Mermaid（但也允许推荐更好的替代）
[强制步骤1] 先准备改进方案和升级对应关系
[强制步骤2] 然后执行修改
[允许动作] 上网搜索最佳实践
[上下文约束] 学术论文、Typst 环境
```

## 通用模式

```
针对 [目标文档] 中的 [元素类型]，分析是否使用 [候选工具] 重绘/重构效果更好。
先准备 [改进方案] 和 [升级对应关系/映射表]，然后 [执行修改]。
（可上网搜索 [技术领域] 在 [平台] 中的最佳实践 / 有什么好用的 skill）
```

## 适用变体

1. **论文公式排版**
   > 针对论文中的数学公式，分析是否使用 unicode-math 或 mathtype 效果更好，先准备排版方案，然后修改。

2. **代码高亮升级**
   > 针对论文中的代码片段，分析是否使用 minted 或 listings 效果更好，先准备方案，然后修改。

3. **表格样式统一**
   > 针对论文中的数据表格，分析是否使用 tablex 或 pandas-styling 效果更好，先准备方案，然后修改。

4. **PPT 图表矢量化的通用版**
   > 针对 PPT 中的架构图，分析是否使用 Mermaid / Draw.io / TikZ 重绘效果更好，先准备方案，然后修改。

## 关键成功因素

- **必须做现状分析**：列出所有现有图表清单、分类、被引用情况
- **必须做工具调研**：搜索候选工具在目标平台上的实际可用性、版本、已知限制
- **必须输出对应关系表**：原图 → 新方案 → 优先级 → 理由
- **必须验证编译**：修改后必须实际编译通过，不能只看代码逻辑
- **必须备份原图**：防止不可逆破坏

## 新增 Prompt 模式：答辩材料反幻觉审稿

### 原始 Prompt
> @slides 里有我们制作的基于md和slidev的毕业设计答辩PPT和讲稿，但是里面内容幻觉严重，请你站在会质疑AIGC的评委的角度给出尖锐的攻击和修改意见，并模拟答辩上发生概率大的Q&A

### Prompt 结构拆解

```
[目标对象] 答辩PPT + 讲稿
[问题判断] 内容幻觉严重 / 包装过度 / 证据链不足
[审查视角] 对AIGC敏感、会追问真实性的评委
[输出1] 尖锐攻击点
[输出2] 可执行修改建议
[输出3] 高频答辩Q&A
[隐含要求] 从“好看”转向“可信”，识别最容易被追问的地方
```

### 通用模式

```
审查 [答辩材料/汇报稿/演示文稿]，重点识别其中
1. 容易被质疑为AIGC生成或包装过度的表述
2. 缺少证据链、来源、实验支撑的结论
3. 会被评委高概率追问的创新点、测试、系统实现细节

请站在 [某类评审角色] 的角度，输出：
- 尖锐攻击点
- 对应修改建议
- 高概率问答与推荐回答
```

### 关键成功因素
- **必须把“攻击点”落到具体页或具体表述**，不能只给泛泛提醒
- **必须区分“功能演示”与“实验结论”**，避免误判
- **必须把“创新”降级检查一遍**：哪些是理论创新，哪些只是工程实现
- **必须审查LLM相关表述**：必要性、边界、降级、评估证据

## 新增 Prompt 模式：生成答辩Q&A速查表与准备手册

### 原始 Prompt
> 好的，而且给我一个类似Q&A应对的cheatsheet，让我可以提前应对，并给我提供一个准备答辩的辅导手册

### Prompt 结构拆解

```
[目标对象1] 答辩Q&A应对cheatsheet
[目标对象2] 答辩准备辅导手册
[使用场景] 毕业答辩前准备 + 现场速查
[核心诉求] 提前应对高频追问，降低临场慌乱
[隐含要求] 不是继续润色正文，而是补“辅助作战材料”
[输出风格] 可背诵、可速查、可按时间执行
```

### 通用模式

```
基于 [答辩PPT/讲稿/项目材料]，为我补充两类辅助文档：
1. 一份高频问答应对 cheatsheet
2. 一份按时间线组织的准备/训练辅导手册

要求：
- cheatsheet 适合现场速查，尽量短、稳、可直接说
- 辅导手册适合答辩前几天系统准备，包含节奏、训练方式、检查清单
- 整体口径以“可信、收敛、能承认边界”为主，而不是夸大成果
```

### 关键成功因素
- **必须区分“速查材料”和“系统准备材料”**，二者不是一回事
- **必须服务真实临场场景**：要考虑用户紧张、时间短、容易忘词
- **必须把口头表达降级为可信表述**，不能继续放大包装感
- **必须给出可执行的时间线和清单**，否则手册很难真正落地

## 新增 Prompt 模式：重绘一张已经“糊成一坨”的复杂流程图

### 原始 Prompt
> 系统核心业务流程图，现在就是一坨，用你觉得合适的方法和工具重新画一下

### Prompt 结构拆解

```
[目标对象] 单张复杂业务流程图
[问题判断] 现有图可读性差 / 信息层次混乱 / 版式像一坨
[工具选择权] 允许助手自行决定最合适的方法与工具
[输出目标] 不是仅改文字，而是直接重绘图稿并接入文档
[隐含要求] 兼顾论文正文阅读性与后续答辩复用
```

### 通用模式

```
定位 [文档/项目] 中的 [目标图表]，判断当前问题属于：
- 信息过多
- 结构层次不清
- 工具不适配
- 分辨率或版式不足

然后自行选择最合适的重绘方式（如 Typst 原生图 / Mermaid / Fletcher / PIL脚本 / 专业工具导出PNG），
完成：
1. 定位原图引用位置
2. 设计新的信息分层与布局方案
3. 生成新图并接入文档
4. 实际编译或渲染验证
```

### 关键成功因素
- **必须先判断“问题是内容还是版式”**，不能一上来就换工具
- **必须按目标阅读场景选工具**：论文正文里复杂泳道图常更适合脚本或专业工具导出 PNG
- **必须做最终落地验证**：不仅要生成图，还要确认正文/PPT中的引用已经切到新图
- **必须考虑资源复用**：若图稿后续可能进答辩PPT，应同步到公共图像目录

## 新增 Prompt 模式：微调流程图的箭头、边框与比例参数

### 原始 Prompt
> @image:image.png 优化箭头和边框的视觉处理效果，确保设计元素的对齐精确、比例协调。具体修复以下方面：
> 1. 调整箭头尖端与连接点的精准对接
> 2. 统一边框线条的粗细和颜色值
> 3. 检查所有转角处的平滑衔接
> 4. 确保响应式布局中各尺寸下的显示一致性
> 请提供可视化修改方案，并说明调整的具体参数值。

### Prompt 结构拆解

```
[目标对象] 已生成但细节粗糙的流程图
[问题层级] 不是重画信息架构，而是做视觉几何精修
[修复重点1] 箭头尖端与连接点对接精度
[修复重点2] 边框粗细和中性色值统一
[修复重点3] 折线转角平滑
[修复重点4] 多尺寸显示一致性
[输出要求] 给出可视化修改方案 + 明确参数值
```

### 通用模式

```
针对 [现有图稿/截图] 做“几何与视觉参数精修”，重点检查：
1. 箭头头部是否与目标锚点精确对接
2. 结构边框是否存在多套相近但不一致的宽度和色值
3. 折线路径的转角是否生硬或出现粗细突变
4. 固定像素偏移是否会在缩放后破坏比例

然后：
- 抽取统一的描边/箭头参数
- 改写绘制逻辑而不只是调几个坐标
- 重新生成图稿并验证实际输出
- 向用户给出“修改项 → 参数值 → 视觉收益”的说明表
```

### 关键成功因素
- **必须把问题拆到几何层**：很多“看起来不顺”的问题，本质是箭头头部、线段终点和锚点没有分离控制
- **必须统一设计 token**：颜色和线宽要先收敛成有限变量，再谈微调
- **必须考虑缩放场景**：论文图、PPT图、网页预览图的阅读距离不同，固定像素值很容易失效
- **必须给出明确参数值**：否则“优化了视觉效果”这种说法不可复用也不可验收

## 新增 Prompt 模式：继续收紧一张已经重绘过但仍显拥挤的流程图

### 原始 Prompt
> 问题不在这里，而是1. Hub-and-Spoke履约链路之间的空隙太窄导致箭头没地方放，太挤了。2. 底下Feign同步、MQ异步、人工确认和说明没有和上面对齐，这一块我觉得可以删掉。总体上，我们要把这张图做的更干净一点，不要乱，你自己看一下画出来的效果，然后刷回论文。

### Prompt 结构拆解

```
[目标对象] 已经重绘过一轮的流程图
[反馈来源] 用户基于成图效果给出二次视觉反馈
[核心问题1] 主链路节点太宽，箭头留白不足
[核心问题2] 底部图例/说明区重复且不对齐
[总体目标] 更干净、更少干扰，而不是增加新元素
[执行要求] 先自己判断效果，再刷回论文
```

### 通用模式

```
针对一张已经初步重绘的图稿，基于用户的二次视觉反馈继续收紧：
1. 判断真正拥挤的是节点、箭头还是冗余说明区
2. 删除重复信息块，而不是默认继续重排它们
3. 通过缩窄关键节点、设置非对称边距、预留箭头走廊来改善主链路可读性
4. 重新生成成图后再人工复看，再刷回正文
```

### 关键成功因素
- **必须相信用户指出的视觉症结**：用户说“问题不在这里”，就要回到成图重新看，而不是执着于之前优化的方向
- **必须优先做减法**：重复图例、底部说明、小标签往往是最该先删的部分
- **必须把留白分配给连接关系**：链路图的阅读重点是流向，不是节点面积平均
- **必须先看成图再下结论**：这类任务不能只盯代码参数，必须结合实际生成效果继续收紧

## 新增 Prompt 模式：把短箭头流程进一步合并成 block

### 原始 Prompt
> @image:image.png 图片还是有点脏，箭头的起点终点不太妥当，箭头的文字说明和其他内容有重叠遮挡， @image:image.png 这里的箭头如果是这样，还不如合并为一个block

### Prompt 结构拆解

```
[目标对象] 已经压缩过一轮但仍显脏的流程图
[问题1] 某些箭头起终点仍不自然
[问题2] 箭头说明标签和节点内容发生遮挡
[问题3] 某些短箭头的信息价值低于其视觉噪音
[关键建议] 对短箭头连接的步骤直接合并成 block
[目标] 从“勉强能看”继续收敛到“论文级干净”
```

### 通用模式

```
当用户指出流程图“还是脏”时，不要只继续微调箭头端点；要重新判断：
1. 哪些箭头只是形式连接，没有足够的信息增量
2. 哪些箭头标签是重复说明，可以删掉
3. 哪些相邻步骤适合合并成一个更大的过程块
4. 哪些折线应统一改为正交走线以提升论文感

然后：
- 合并短链路步骤为 block
- 删除易重叠的补充标签
- 只保留最必要的调用语义
- 重新生成成图并人工复看
```

### 关键成功因素
- **必须承认“箭头太短”本身就是设计问题**，而不是总试图把短箭头修漂亮
- **必须从信息价值判断是否合并**：如果箭头不提供额外可读性，就应让位于更大的过程块
- **必须优先消除遮挡**：文字与边线冲突时，删除说明通常比继续挪坐标更有效
- **必须以成图观感为最终准绳**：流程图是否干净，只有看最终图才知道

## 新增 Prompt 模式：把彩色流程图收敛为黑白稿并继续修正箭头指向

### 原始 Prompt
> 目前箭头的指向还有问题，优化一下，然后把颜色去掉，只留下黑白轮廓和边框

### Prompt 结构拆解

```
[目标对象] 已完成多轮调整的流程图
[问题1] 箭头起点/终点的方向感仍不自然
[问题2] 彩色泳道和色块已经不再服务阅读，反而分散注意力
[视觉目标] 收敛成论文/打印友好的黑白轮廓图
[执行要求] 不是只改配色，还要同步修正关键箭头的入射方式
```

### 通用模式

```
针对一张已经基本定型的流程图，继续做“最终收敛版”处理：
1. 检查哪些箭头仍存在指向不自然、先拐后进、落点别扭的问题
2. 通过按目标坐标反推锚点，让关键箭头尽量沿同一坐标轴进入目标框
3. 将整张图从分彩稿收敛为黑白稿：主轮廓、分隔线、填充三层即可
4. 重新生成成图并确认论文中的最终引用已刷新
```

### 关键成功因素
- **必须把“箭头方向问题”理解为路径几何问题**，而不是只继续微调箭头三角形
- **必须按黑白稿重新设计层次**：去颜色不等于把原来的浅蓝浅紫全部换成浅灰
- **必须保留必要的阅读层级**：黑白图也需要主轮廓、辅助分隔、留白三种角色
- **必须重新渲染并看最终图**：黑白稿是否干净，只看代码改动是不够的

## 新增 Prompt 模式：只微调单条箭头，不扰动已经接近完成的图稿

### 原始 Prompt
> logisticsService到 写入调度池 这个箭头可以微调成先向右再向下，其他的不用动，已经趋于完美

### Prompt 结构拆解

```
[目标对象] 已接近定稿的流程图
[修改范围] 只允许动一条指定箭头
[用户态度] 整体已经满意，只要局部修饰
[执行约束] 不要顺手继续优化别的地方
[几何目标] 把单条箭头改成指定走向（先右后下）
```

### 通用模式

```
当用户明确说“其他都不用动”时：
1. 锁定唯一目标元素
2. 只调整该元素的几何路径，不改节点尺寸、布局和其它连线
3. 重新生成最终图并确认没有意外扰动
4. 以“局部微调完成”而不是“大版本优化”收尾
```

### 关键成功因素
- **必须尊重“不要再动别处”这个约束**，不要趁机继续优化其它细节
- **必须让折线路径变化可见但克制**：只要用户能感知顺眼了就够，不需要大改
- **必须重新渲染验证**：单条箭头微调也可能影响整体观感
